# 솔루션 비교

::: warning 번역 안내
이 페이지는 [중국어 원문](../../guide/comparison.md)을 기준으로 정리했습니다. 차이가 있을 경우 원문을 우선합니다.
:::

이 문서는 모메이와 WordPress, Hugo, Hexo를 기술 스택, 국제화, 운영 편의성, 확장성 관점에서 비교합니다. 모메이의 핵심 차별점은 **AI + 다국어 + 운영 백오피스**를 하나의 제품 흐름으로 묶는 데 있습니다.

## 1. 비교 포인트

| 항목 | Momei | WordPress | Hugo | Hexo |
| :--- | :--- | :--- | :--- | :--- |
| 기술 스택 | Nuxt + TypeScript | PHP | Go | Node.js |
| 렌더링 | SSR / SSG / ISR | 동적 렌더링 | 정적 생성 | 정적 생성 |
| 국제화 | 원생 통합 | 플러그인 의존 | 직접 구성 | 플러그인 / 수동 |
| 작성 흐름 | Markdown + AI | 시각 편집기 중심 | Markdown | Markdown |
| 관리자 화면 | 내장 현대식 UI | 성숙하지만 무거움 | 별도 구성 필요 | 별도 구성 필요 |

## 2. 모메이의 장점

- 다국어 콘텐츠와 번역 클러스터를 제품 차원에서 관리
- AI 제목, 요약, 번역, Slug, 음성 초안까지 통합
- 설정, 알림, 저장소, 배포 가이드를 한 시스템 안에서 연결
- Slug 보존을 통한 SEO 친화적 마이그레이션

## 3. 어떤 경우에 적합한가

- 다국어 기술 블로그를 운영하고 싶을 때
- AI 기반 작성 워크플로를 실제 운영에 쓰고 싶을 때
- 현대적인 Nuxt 생태계와 자체 확장성을 동시에 원할 때
