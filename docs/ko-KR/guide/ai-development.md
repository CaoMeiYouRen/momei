# AI 협업 개발 가이드

::: warning 번역 안내
이 페이지는 [중국어 원문](../../guide/ai-development.md)을 기준으로 정리했습니다. 차이가 있을 경우 원문을 우선합니다.
:::

모메이는 AI를 단순한 코드 생성기가 아니라, 계획 수립부터 구현, 검증, 문서 동기화까지 함께 수행하는 협업 주체로 다룹니다. 이 가이드는 사람이 의도를 정의하고 AI가 반복 작업을 맡는 방식으로 프로젝트를 다루는 방법을 설명합니다.

## 1. 핵심 개념

- 단순 수정은 수동으로 처리할 수 있습니다.
- 기능 개발, 버그 수정, 대규모 리팩터링, 테스트 보강은 AI 에이전트와의 협업이 더 효율적입니다.
- 중요한 것은 AI에게 일을 넘기는 것이 아니라, 올바른 맥락과 제약을 제공하는 것입니다.

## 2. 에이전트 체계

프로젝트에는 역할별 에이전트가 준비되어 있습니다. 예를 들면:

- 기능 구현: `@full-stack-master`
- 테스트 강화: `@test-engineer`
- 질의 응답: `@qa-assistant`
- 코드 리뷰: `@code-reviewer`

역할에 따라 읽을 문서와 허용된 작업이 다르므로, 먼저 [AGENTS.md](../../AGENTS.md)를 확인하는 것이 좋습니다.

## 3. PDTFC+ 흐름

1. **Plan**: 요구사항, todo, 관련 표준 문서를 읽고 범위를 확정합니다.
2. **Do**: 실제 변경을 수행합니다.
3. **Test**: lint, typecheck, 관련 테스트 또는 문서 사이트 빌드를 실행합니다.
4. **Fix**: 검증 중 나온 문제를 정리하고 수정합니다.
5. **Enhance / Finish**: 문서, 테스트, todo 상태, 전달 메모를 보강합니다.

## 4. 협업 팁

- 요구사항이 애매하면 먼저 인터뷰 모드로 의도를 분해합니다.
- 검증 가능한 문서와 파일 경로를 직접 제공하면 결과 품질이 올라갑니다.
- 최종 코드는 반드시 사람이 아키텍처와 위험을 검토해야 합니다.

## 5. 다른 AI 도구와의 호환

- GitHub Copilot 중심으로 설계되어 있지만 Claude Code, Cursor 류 도구에도 적용할 수 있습니다.
- 핵심은 에이전트 규칙, PDTFC+ 흐름, 문서 우선 접근 방식을 유지하는 것입니다.
