# 계획 및 평가 표준

::: warning 번역 안내
이 페이지는 [중국어 원문](../../standards/planning.md)과 [영문판](../../en-US/standards/planning.md)을 바탕으로 정리했습니다. 차이가 있을 경우 중국어 원문을 우선합니다.
:::

## 1. 목표

프로젝트 계획은 ROI, 위험, 인수 조건, 단계별 폐쇄 루프를 함께 고려해야 합니다.

## 2. 주요 문서

- `roadmap.md`: 중장기 방향과 단계 목표
- `todo.md`: 현재 단계의 실행 항목
- `todo-archive.md`: 완료 및 보관된 항목

## 3. 실무 규칙

- 새로운 요구는 먼저 roadmap에 기록하고 이후 todo로 분해합니다.
- 각 릴리스 후 상태와 인수 기준을 함께 갱신합니다.
