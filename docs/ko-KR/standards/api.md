# API 표준

::: warning 번역 안내
이 페이지는 [중국어 원문](../../standards/api.md)과 [영문판](../../en-US/standards/api.md)을 바탕으로 정리했습니다. 차이가 있을 경우 중국어 원문을 우선합니다.
:::

## 1. 핵심 요구사항

- API 응답 구조는 일관되어야 합니다.
- 라우트 명명, 검증, 권한 체크는 공통 규칙을 따라야 합니다.
- 공개 API와 관리자 API의 경계는 분명해야 합니다.

## 2. 점검 포인트

- schema / type으로 인터페이스 경계가 정의되어 있는지
- 오류 코드와 메시지가 일관적인지
- 필요한 인증과 권한 검사가 누락되지 않았는지
