# 보안 개발 표준

::: warning 번역 안내
이 페이지는 [중국어 원문](../../standards/security.md)을 기준으로 정리했습니다. 차이가 있을 경우 원문을 우선합니다.
:::

이 문서는 인증, 데이터 처리, 웹 보호, 의존성, CLI 자동화 측면의 보안 요구사항을 정리합니다.

## 1. 인증과 권한

- 사용자 데이터가 관련된 API는 반드시 세션 검증을 거쳐야 합니다.
- 다중 역할 판단은 공통 권한 함수로 처리합니다.
- 비밀번호는 절대 평문 저장하지 않습니다.

## 2. 데이터 보안

- 모든 입력은 schema 검증을 거칩니다.
- SQL 문자열 연결을 피하고 ORM / 파라미터 바인딩을 사용합니다.
- 응답 전 민감 데이터를 마스킹하거나 제거합니다.
- 비밀값은 환경 변수로만 관리합니다.

## 3. 웹 보호와 로그

- `v-html` 같은 고위험 렌더링 포인트는 별도 감사가 필요합니다.
- 운영 환경에서 무분별한 CORS 허용은 금지합니다.
- 중요한 작업은 감사 로그를 남기되, 로그에 비밀값을 출력하지 않습니다.

## 4. 자동화 안전

- 셸 명령 전 OS와 환경을 확인합니다.
- 삭제나 덮어쓰기 전 경로를 검증합니다.
- 빈 경로나 위험한 와일드카드로 삭제를 실행하지 않습니다.
