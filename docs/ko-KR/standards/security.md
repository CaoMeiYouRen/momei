# 보안 개발 표준

::: warning 번역 안내
이 페이지는 [중국어 원문](../../standards/security.md)과 [영문판](../../en-US/standards/security.md)을 바탕으로 정리했습니다. 차이가 있을 경우 중국어 원문을 우선합니다.
:::

## 1. 기본 원칙

- 민감한 값은 코드에 하드코딩하지 않습니다.
- 환경 변수, 권한 체크, 관리자 기능은 명확히 분리합니다.
- 스크립트와 자동화 명령은 실행 전 경로와 환경을 점검합니다.

## 2. 고위험 영역

- 인증과 권한
- 관리자 설정과 민감 API
- 파일 업로드, 이메일, 외부 서비스 키
