# 문서 표준

::: warning 번역 안내
이 페이지는 [중국어 원문](../../standards/documentation.md)을 기준으로 정리했습니다. 차이가 있을 경우 원문을 우선합니다.
:::

이 문서는 모메이 문서의 작성, 구조화, 번역 유지 규칙을 설명합니다. 문서는 사람뿐 아니라 AI 에이전트의 작업 품질에도 직접 영향을 줍니다.

## 1. 문서 구조

- `guide/`: 시작, 배포, 개발, AI 협업 안내
- `standards/`: 개발, 테스트, 보안, 계획, 문서 표준
- `design/`: UI, API, 데이터베이스 등 상위 설계
- `plan/`: 로드맵, todo, 아카이브
- 각 번역 로케일 디렉터리는 원문 구조를 최대한 유지합니다.

## 2. 작성 규칙

- 문서마다 H1은 하나만 사용합니다.
- 제목 계층은 순차적으로 사용합니다.
- 내부 문서 링크는 상대 경로를 사용합니다.
- 복잡한 구조는 표, 목록, Mermaid로 표현하는 것이 좋습니다.

## 3. 번역 페이지 규칙

- `source_branch`, `last_sync`를 넣어야 합니다.
- 상단에 중국어 원문이 우선이라는 번역 안내를 둡니다.
- nav/sidebar와 함께 유지해야 합니다.

## 4. 유지보수 원칙

- 새 페이지를 추가하면 문서 사이트 설정을 함께 갱신합니다.
- 코드 변경 후 관련 설계 / 가이드 / README가 함께 수정되어야 할지 점검합니다.
- 명시적 요청 없이는 `CHANGELOG.md`를 수정하지 않습니다.
