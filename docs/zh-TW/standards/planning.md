# 規劃與評估規範

::: warning 翻譯說明
本頁對應 [中文原文](../../standards/planning.md) 與 [英文版本](../../en-US/standards/planning.md)。若內容有差異，請以中文原文為準。
:::

## 1. 規劃目標

專案規劃需同時兼顧 ROI、風險、驗收條件與階段閉環，避免任務只啟動不收斂。

## 2. 常用規劃物件

- `roadmap.md`：中長期規劃與階段目標
- `todo.md`：目前階段的落地執行項
- `todo-archive.md`：已完成與已歸檔任務

## 3. 實作要求

- 新需求先進 roadmap，再拆到 todo。
- 每輪交付要同步更新狀態與驗收描述。
- 涉及多語言或工程治理時，要反映到規劃文檔。
