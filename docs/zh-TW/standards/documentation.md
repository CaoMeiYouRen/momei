# 文件規範

::: warning 翻譯說明
本頁對應 [中文原文](../../standards/documentation.md)。若內容有差異，請以中文原文為準。
:::

本文檔定義墨梅文檔的撰寫、組織與維護標準，目的是確保文檔同時服務於人類開發者與 AI 智能體。

## 1. 目錄結構

所有文檔應存放於 `docs/`，並依用途分類：

- `guide/`：快速開始、部署、開發與 AI 協作指南
- `standards/`：開發、測試、安全、規劃與文件規範
- `design/`：高層架構、UI、API、資料庫與模組設計
- `plan/`：路線圖、待辦與歸檔
- `en-US/`、`zh-TW/`、`ko-KR/`：翻譯文檔目錄

## 2. 撰寫要求

- 每個文件只能有一個 H1。
- 標題層級應依序遞進，不要跳級。
- 優先使用表格、列表與 Mermaid 描述結構與流程。
- 站內引用必須使用相對路徑。

## 3. 翻譯頁要求

- 翻譯頁需包含 `source_branch` 與 `last_sync`。
- 頁首需保留翻譯說明，聲明中文原文為準。
- 英文、繁中、韓文等翻譯目錄應盡量與中文原文保持相同檔名與結構。

## 4. 維護與站點配置

- 新增頁面後，需同步更新 VitePress 的導航與側邊欄。
- 修改核心功能後，應主動檢查是否需要同步更新指南、設計與 README。
- 除非明確要求，禁止手動修改 `CHANGELOG.md`。

## 5. 國際化策略

- 對外關鍵文檔應優先翻譯。
- 快速變動的模組設計可暫以中文原文為準。
- 路線圖可採漸進式翻譯，但已完成階段應盡量提供完整可讀版本。
