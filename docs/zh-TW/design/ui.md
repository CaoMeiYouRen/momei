# UI 設計

::: warning 翻譯說明
本頁對應 [中文原文](../../design/ui.md)。若內容有差異，請以中文原文為準。
:::

墨梅的 UI 設計靈感來自「墨」與「梅」的東方氣質，同時結合現代 Web 應用對可讀性、效率與沉浸感的要求。這份文檔用於描述整體風格、主題系統、交互原則與管理後台常用組件方向。

## 1. 設計理念

- **簡潔**：讓內容本身成為視覺主體，減少噪音。
- **現代**：使用恰到好處的動效與狀態反饋，而不是過度炫技。
- **沉浸**：閱讀與創作場景都應避免干擾。
- **隱私與低打擾**：只有在必要互動時才提升驗證或登入門檻。

## 2. 視覺系統

### 2.1 主題與色彩

- 基於 PrimeVue Aura 主題做定制。
- 支援淺色、深色與跟隨系統模式。
- 核心顏色包含主色、點綴色、表面色與文本色，且可由管理端配置。

### 2.2 字體策略

- 中文優先採用系統無襯線字體。
- 英文與介面字體可使用更現代的無襯線家族。
- 程式碼區域應使用獨立等寬字體。

### 2.3 主題覆蓋

- 透過 CSS Variables 與分層覆蓋 PrimeVue 樣式。
- 主題切換應避免 FOUC，並保有平滑過渡。

## 3. 組件與版面

- 以 PrimeVue 作為基礎組件庫。
- 按鈕、卡片、輸入框等元件需保持一貫的狀態回饋。
- 內容區域寬度需適合長文閱讀。
- 管理端應優先抽象通用頁首、確認對話框、表格與翻譯編輯器等能力。

## 4. 響應式與移動端

- 設計採 Mobile First。
- 移動端需確保導航、點擊區域與文章正文排版都足夠可讀。
- 長程式碼區塊、表格與圖片需要有合理的溢出與縮放處理。

## 5. 動效與體驗

- 頁面切換以淡入淡出或輕微位移為主。
- Loading 優先使用骨架屏而非無止盡轉圈。
- 微交互應為理解狀態服務，例如按鈕 hover、切換動畫與列表漸入。

## 6. 維護原則

- UI 變更應與設計語言一致，不要引入明顯割裂的新風格。
- 前台與管理端都應遵循相同的品牌基調。
- 若改動涉及關鍵交互或布局，應同步安排 UI 驗證。
