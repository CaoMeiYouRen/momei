# API 設計

::: warning 翻譯說明
本頁對應 [中文原文](../../design/api.md) 與 [英文版本](../../en-US/design/api.md)。若內容有差異，請以中文原文為準。
:::

## 1. 設計重點

- 區分公開端與管理端 API。
- 使用一致的資料結構、錯誤回應與驗證方式。
- 與權限、設定與通知等系統能力保持一致。
