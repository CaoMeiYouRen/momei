# 墨梅博客 长期规划与积压项 (Backlog)

本文档用于维护尚未进入正式阶段执行面的统一候选池，并按“长期主线任务”与“短期 / 一次性候选任务”双轨区分。当前阶段执行面请参阅 [项目计划](./roadmap.md)、[待办事项](./todo.md) 与 [待办归档](./todo-archive.md)。

> **维护规则**:
> 1. 新功能需求、非阻塞优化与长期治理事项优先写入本文件，而不是直接写入 `todo.md`。
> 2. backlog 必须区分“长期主线任务”和“短期 / 一次性候选任务”：长期主线可跨阶段保留，短期 / 一次性候选在正式上收后必须去重。
> 3. 长期主线被某阶段抽取后，不删除主线卡片，只补记最近一次上收阶段、当前状态与下一次可切片方向。
> 4. 当前仓库的 backlog 以中文为唯一事实源；翻译文档只保留摘要或跳转说明。

## 短期 / 一次性候选任务（上收后去重）

> 共享说明：除非单项另有说明，本区块条目当前均处于“候选评估中”，默认尚未满足正式上收前置条件；只有当条目内约束、门槛或预研结论闭环后，才允许写入 roadmap / todo。

1. **README 多语言镜像存放位置重评估**
- **目录方案评估**:
    - 评估是否将 `README.<locale>.md` 迁移到 `docs/` 下的专门目录（如 `docs/readme/` 或 `docs/i18n/<locale>/readme/`），但必须先明确 GitHub 仓库入口、外部引用与文档站导航是否仍需保留根目录镜像或兼容跳板。
- **规范与链接同步**:
    - 若目录调整成立，需要同步修正文档规范、VitePress 导航、README 互跳链接、翻译治理说明与外部链接映射，避免出现“仓库入口一套、文档站一套”的双重事实源。
- **迁移门槛**:
    - 在未证明不会削弱 GitHub 首屏可读性、不会破坏现有多语言 README 入口之前，该事项维持 backlog 评估，不直接纳入当前阶段执行。

2. **AI 初始化 / 问答助手评估与工程复用治理**
	- 验收: 评估是否提供面向新部署者的 AI 初始化助手或配置问答助手，先收敛可回答范围、可写配置范围、ENV 锁定字段、越权保护与失败回退策略，再决定是否立项实现。
	- 验收: 若进入实现阶段，应优先复用现有安装向导、Settings 元信息、`qa-assistant` 与文档知识源，而不是额外维护第二套配置事实源。

3. **`caomei-auth` 第三方登录支持评估与接入预研**
- **协议与能力基线**:
    - 评估 `caomei-auth` 是否提供可被 `better-auth` / `genericOAuth` 稳定消费的 OAuth 2.1 / OIDC 能力，至少包括授权地址、令牌地址、用户信息地址或等效 discovery 信息，以及 `state`、PKCE、`nonce`、错误码和 scope 约束。
    - 明确登录、刷新、登出、令牌撤销、JWK / 签名校验与多环境回调地址的支持边界，避免接入后只能“能登录一次”，却无法稳定维护会话生命周期。
- **账户映射与资料契约**:
    - 明确 `caomei-auth` 返回的稳定用户标识、`email` / `email_verified`、昵称、头像、locale 等字段契约，确保与现有 `better-auth` 用户模型、账号绑定与多语言界面兼容。
    - 评估同邮箱合并、已存在账号绑定、第三方账号解绑与异常回退策略，避免引入重复账号或破坏现有认证链路。
- **配置、安全与反向需求输出**:
    - 梳理接入所需的 Client ID / Secret、回调 URL、权限范围、品牌文案、图标、合规提示与 ENV 锁定策略，确认哪些配置必须继续维持部署期环境变量管理。
    - 基于接入过程反推 `caomei-auth` 需要补齐的能力，例如 discovery 文档、标准 claims、刷新令牌、撤销接口、JWK 暴露、租户隔离或审计字段，再决定是否正式排期。

4. **桌面端应用 (Desktop Application)**
- **Tauri 跨平台应用**:
    - 实现桌面客户端骨架，支持单站点/多站点管理。
    - 支持离线 Markdown 写作与间断性云端同步功能。
    - 提供原生菜单和系统集成，提升桌面体验。

5. **极客技术增强 (Geek Tech Extras)**
- **可执行代码块支持**:
    - 实现 Markdown 代码块在特定环境下的运行与结果输出。
    - 支持常见编程语言的代码执行（如 JavaScript、Python、Shell）。

6. **主题生态系统 (Theme Ecosystem)**
- **主题社区与发布平台**:
    - 允许创作者发布、分享并由他人安装自定义主题。
    - 建立安全审核机制防范 XSS 攻击。
- **画廊与市场**:
    - 支持保存多套自定义配置，建立主题预览、发布与管理中心。
    - 提供主题评分、评论和下载统计。
- **安全性与性能**:
    - 实现针对发布主题的自动化 XSS 扫描。
    - 使用 CSS Layer 分层策略提升主题性能。

7. **商业化与会员体系 (Monetization & Membership)**
- **支付体系集成**:
    - 集成 Stripe/支付宝支付接口。
    - 支持打赏功能或 Pro 会员准入策略。

8. **播客与多媒体扩展 (Podcast & Multimedia)**
- **全站沉浸式播放**:
    - 全站悬浮播放器，支持跨页面断点续播与内容同步。

9. **AI 视频生成与增强 (AI Video Generation & Enhancement)**
- **多模态内容产出**:
    - 探索集成视频生成模型（如 Seedance 2.0、~~Sora 2.0~~ 等），支持基于文章内容或脚本生成动态视频素材。
    - 实现“文章转视频”工作流，为技术博文自动生成短视频概览或演示。
