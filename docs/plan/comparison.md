# 方案对比 (Comparison)

本页将墨梅 (Momei) 与目前主流的博客方案进行全方位对比，帮助你了解墨梅的独特优势。

## 1. 核心对比表

| 特性              | 墨梅 (Momei)              | WordPress          | Hugo               | Hexo                |
| :---------------- | :------------------------ | :----------------- | :----------------- | :------------------ |
| **技术栈**        | Nuxt 3 (TypeScript)       | PHP                | Go                 | Node.js             |

## 2. 墨梅的独特优势

### 2.1 原生多语言支持

大多数静态博客和经典 CMS 的国际化通常是“后加”的功能，往往需要复杂的路由配置或依赖重量级插件。墨梅从底部设计之初就考虑了国际化，支持：

-   内容的多语言平行创作。
-   前端 UI 的一键切换。
-   基于浏览器语言的智能重定向。

### 2.2 现代化技术栈的灵活性

依托 Nuxt 3，墨梅不仅拥有静态博客的快，还拥有 SSR (服务端渲染) 的强和 ISR (增量静态再生) 的便。你可以在不重新发布整个站点的情况下更新某篇文章。

### 2.3 开发者的友好度

对于熟悉 Vue 或 TypeScript 的开发者，墨梅的代码库非常整洁且易于扩展。内置的 AI 代理 (Agents) 支持不仅方便了内容的自动化生成，也降低了文档维护的门槛。

### 2.4 SEO 平滑迁移

通过支持 `Slug` (路径别名)，墨梅允许你完全自定义每一篇文章的路径。这意味着你可以从 WordPress、Hexo 等任何平台迁移过来，而不需要改变原本的链接，从而保护了多年的 SEO 积累。

### 2.5 灵活的主题自定义 (规划中)

为了让博客更符合个人审美，墨梅计划引入强大的主题引擎。区别于传统的硬编码样式，我们将：

-   **解耦结构与表现**：核心业务逻辑与 UI 表现层分离，允许用户像使用 Hexo 一样更换“外壳”。
-   **自定义配置注入**：支持通过管理后台直接注入 CSS 变量或 SCSS 片段，无需重新打包即可改变配色与布局细节。
-   **开发者友好**：为高级用户提供主题插槽和生命周期钩子，方便进行深度二次开发。

## 3. 该如何选择？

-   **如果你需要：** 极速的加载体验、原生多语言支持、完全的代码控制力、以及现代化的开发体验。
    -   **👉 选择 墨梅 (Momei)**
-   **如果你需要：** 海量的成熟插件、极其丰富的社区模板、且不需要在意服务器维护开销。
    -   **👉 选择 WordPress**
-   **如果你需要：** 极致的编译速度、且对 Go 模板语言非常熟悉。
    -   **👉 选择 Hugo**
-   **如果你需要：** 庞大的中文插件生态、且习惯使用传统的 Node.js 静态生成。
    -   **👉 选择 Hexo**
