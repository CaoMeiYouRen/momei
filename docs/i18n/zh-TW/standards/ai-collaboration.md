# AI 協作規範

本規範定義了墨梅專案中 AI 智能體的基本行為準則，目的是確保 AI 在自動化過程中保持邏輯嚴密、安全合規，且不破壞既有工程品質。

## 1. 身份認同

每個 AI 智能體在啟動任務前，都必須先明確自己的角色定位與權限邊界。

- **禁止跨權**：只讀型助手不可修改代碼；測試型角色應聚焦測試補齊與驗證。
- **環境感知**：執行命令前必須確認當前作業系統與終端環境，避免命令語法錯誤。
- **文檔感知**：進入實作前應先讀取對應規範、設計文檔與待辦內容。

## 2. 核心工作流：PDTFC+

所有寫操作任務都應遵循以下節奏：

1. **Plan**：先閱讀 `todo.md` 與相關文檔，必要時透過提問澄清需求。
2. **Do**：按既定範圍修改程式碼或文檔，避免無關重構。
3. **Audit**：檢查安全邊界、權限判斷、邏輯一致性與敏感資訊處理。
4. **Validate / Test**：執行 lint、typecheck、定向測試或 UI / 文檔站驗證。
5. **Finish**：補齊文件、更新待辦狀態，必要時補充測試與交付說明。

## 3. 安全紅線

1. **敏感文件保護**：除非有明確指示，不得修改 `.env`、`AGENTS.md` 等核心文件。
2. **密鑰脫敏**：不得在程式碼、文件或日誌中硬編碼 API Key、Token 與密碼。
3. **刪除前校驗**：執行刪除命令或覆蓋文件前，必須先確認目標路徑與範圍。
4. **循環上限**：若同一問題連續多次修復仍無法收斂，應停止並請求人類介入。

## 4. 溝通與透明度

- 回覆應直接、專業，避免無意義的包裝語。
- 在提出結論時，應盡量引用具體檔案或規範依據。
- 執行大規模修改前，應先讓使用者知道預計影響的區域。
