# 規劃與評估規範

本文檔用於規範墨梅的階段規劃、需求准入與任務維護方式，目的是避免範圍蔓延，並確保團隊資源持續集中於核心價值。

## 1. 規劃約束

- 每個階段的核心任務應控制在可管理範圍內。
- 新功能不得破壞既有核心能力，例如認證、文章顯示與 SEO。
- 規劃新功能時應同步考慮測試方案與驗收標準。
- 模糊需求在未完成澄清前，不應直接進入實作。

## 2. 評估方法

專案使用 Value、Alignment、Difficulty、Risk 四個維度評估功能優先級：

$$Score = \frac{Value + Alignment}{Difficulty + Risk}$$

使用原則：

- 分數高的功能優先進入當前或下一階段
- 中間區間進入 backlog
- 分數偏低的需求應延後或取消，除非屬於安全修復

## 3. 路線圖與待辦

- `roadmap.md`：中長期規劃與階段目標
- `todo.md`：當前階段的具體執行項
- `todo-archive.md`：已歸檔的歷史任務

新增功能通常應先進入 roadmap，再根據資源與優先級拆入 todo。

## 4. 不可逆原則

已標記完成的階段，不應回頭修改其定義與驗收標準；若發現缺口，應以新任務或增強項處理，而不是回寫歷史。

## 5. 歸檔與閉環

- 當階段核心任務完成後，應將詳細執行內容歸檔。
- `todo.md` 應保持聚焦，只保留正在進行的工作。
- 每輪交付結束後，需同步更新路線圖與相關設計文檔。
