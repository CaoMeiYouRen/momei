# 安全開發規範

本文檔定義墨梅在認證、資料處理、Web 防護、依賴管理與 CLI 自動化方面的安全要求。

## 1. 身份驗證與授權

- 涉及使用者資料的 API 必須經過 session 校驗。
- 多角色場景下，權限判斷應使用語意化工具函數，不要直接寫死等值比較。
- 密碼不得明文保存，應依託 Better Auth 的安全機制。

## 2. 資料安全

- 所有輸入都應經過 schema 驗證。
- 禁止拼接 SQL 字串，應使用 ORM 或參數化方式。
- API 返回前需適度脫敏，避免暴露敏感資訊。
- 所有密鑰與密碼應存放於 `.env` 或宿主環境，而非硬編碼。

## 3. Web 防護

- `v-html` 等高風險渲染點必須額外審計。
- 生產環境不得隨意放開 CORS。
- 應確保 CSRF 或 SameSite Cookie 策略得到合理配置。

## 4. 日誌與依賴安全

- 登入、刪除、權限變更等重要操作應記錄審計日誌。
- 日誌中不得出現密碼、Token 等敏感資料。
- 新依賴需評估必要性，並關注安全公告。

## 5. 終端與自動化安全

- 執行任何腳本前，先確認作業系統與 shell 環境。
- 執行刪除操作前，必須確認目標路徑存在且正確。
- 避免使用空路徑、未定義變數或高風險通配符作為刪除參數。
