# 文件規範

::: warning 翻譯說明
本頁對應 [中文原文](../../../standards/documentation.md)。若內容有差異，請以中文原文為準。
:::

本文檔定義墨梅文件的撰寫、組織、翻譯與維護標準，確保文件同時服務於人類開發者與 AI 智能體。

## 1. 目錄結構

所有文件都應存放於 `docs/`，並依用途分類：

- `docs/guide/`：快速開始、部署、開發與 AI 協作指南
- `docs/standards/`：開發、測試、安全、規劃與文件規範
- `docs/design/`：高層架構、UI、API、資料庫與模組設計
- `docs/plan/`：路線圖、待辦、backlog 與歸檔
- `docs/i18n/<locale>/`：翻譯文件的實體存放路徑

補充約束：

- 對外文檔站 URL 仍維持 `/<locale>/...`，不得把 `i18n/` 暴露為公開路徑。
- 遷移期間可暫時保留舊的 `docs/<locale>/`，但同一公開路由只能對應一份真實源文件。

## 2. 撰寫要求

- 每個文件只能有一個 H1。
- 標題層級應依序遞進，不要跳級。
- 站內引用必須使用相對路徑。
- 複雜結構與流程優先用表格、列表與 Mermaid 表達。

## 3. 翻譯頁要求

- 翻譯頁需包含 `source_branch` 與 `last_sync`。
- 頁首需保留翻譯說明，聲明中文原文為準。
- 翻譯頁應在 `docs/i18n/<locale>/` 下盡量保持與中文原文相同的檔名與結構。
- 單頁從舊的 `docs/<locale>/` 遷移時，必須同步刪除舊副本，並更新 VitePress rewrites 與 editLink。

## 4. 維護與站點配置

- 新增頁面後，需同步更新 VitePress 的導航與側邊欄。
- 目錄遷移時，必須同時維持 URL 相容性與正確的「編輯此頁」來源路徑。
- 除非明確要求，禁止手動修改 `CHANGELOG.md`。
