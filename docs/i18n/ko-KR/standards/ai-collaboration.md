# AI 협업 표준

::: warning 번역 안내
이 페이지는 [중국어 원문](../../../standards/ai-collaboration.md)을 기준으로 정리했습니다. 차이가 있을 경우 원문을 우선합니다.
:::

이 문서는 모메이 프로젝트에서 AI 에이전트가 따라야 할 기본 행동 원칙을 정의합니다. 목표는 AI가 자동화 작업을 수행하더라도 보안, 품질, 문서 정합성을 유지하는 것입니다.

## 1. 역할 인식

- 에이전트는 작업 전 자신의 역할과 권한 경계를 확인해야 합니다.
- 읽기 전용 에이전트는 코드를 수정하지 않습니다.
- 명령을 실행하기 전 현재 OS와 셸 환경을 확인해야 합니다.

## 2. PDTFC+ 흐름

모든 쓰기 작업은 다음 순서를 따릅니다.

1. Plan: todo와 관련 문서를 읽고 범위를 정합니다.
2. Do: 실제 변경을 수행합니다.
3. Audit / Validate: 보안, 권한, 구현 일관성을 점검합니다.
4. Test: lint, typecheck, 관련 테스트나 문서 빌드를 실행합니다.
5. Finish: 문서와 todo 상태를 보강하고 결과를 정리합니다.

## 3. 보안 레드라인

- `.env`, `AGENTS.md` 같은 핵심 파일은 명시적 지시 없이는 건드리지 않습니다.
- 비밀키, 토큰, 비밀번호를 코드나 로그에 남기지 않습니다.
- 삭제나 대규모 덮어쓰기 전에는 경로와 대상 범위를 확인합니다.
- 같은 문제를 반복 수정해도 수렴하지 않으면 사람의 판단을 요청합니다.

## 4. 커뮤니케이션 원칙

- 불필요한 장식 없이 직접적으로 설명합니다.
- 결론에는 근거가 되는 파일, 문서, 규칙을 연결합니다.
- 큰 범위의 수정 전에는 어떤 영역을 건드릴지 먼저 드러내야 합니다.
