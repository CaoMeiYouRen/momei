# Momei 프로젝트 로드맵

::: warning 번역 안내
이 페이지는 [중국어 원문](../../../plan/roadmap.md)을 기준으로 정리했습니다. 계획 문서는 자주 변경되므로 차이가 있을 경우 원문을 우선합니다.
:::

이 문서는 모메이의 중장기 방향을 요약합니다. 현재 단계의 실제 작업 항목은 [todo](../../../plan/todo.md)와 [todo archive](../../../plan/todo-archive.md)를 함께 보아야 하며, 장기 backlog 는 중국어 [backlog 문서](../../../plan/backlog.md)에서 별도로 관리합니다.

## 1. 프로젝트 개요

- 목표: **AI 기반, 네이티브 국제화 개발자 블로그 플랫폼** 구축
- 핵심 가치: 다국어 콘텐츠 운영, AI 보조 창작, 현대적 프론트엔드 / 백엔드 구조, 마이그레이션 친화성
- 대상 사용자: 개발자, 기술 작성자, 글로벌 독자층을 겨냥한 창작자

## 2. 완료된 단계 요약

- 초기 MVP, 콘텐츠 관리, 기본 i18n, 관리자 기능 확보
- AI 요약 / 제목 / 번역, SEO, 구독 체계 강화
- 주제 시스템, 렌더링 강화, 설치 마법사, CLI 마이그레이션, TTS / ASR, 설정 / 스토리지 정합성 보강

## 3. 최신 진행 상황과 다음 단계

- **14-15단계는 감사 후 보관 완료**: 다국어 설정 거버넌스, 법률 문서 검토 흐름, Cloudflare 배포 문구 정정, AI 거버넌스 사실원, 검증 매트릭스, 주기적 회귀 템플릿, 문서 `i18n` 디렉터리 마이그레이션, `ja-JP` 1차 `ui-ready` 도입이 모두 정리되었습니다.
- **15단계 감사 보강**: `.claude/skills/git-flow-manager`의 잔여 빈 디렉터리를 정리했고, `pnpm ai:check`가 미러 쪽의 추가 파일 / 디렉터리도 감지하도록 확장되었습니다.
- **16단계 계획 초점**:
	- 표준 문서 사실원 정리, 중복 규칙 축소, 오래된 API / 보안 / 성능 규격 교정
	- Review Gate 증적화, 다회차 review 고정, 변경 유형별 검증 명령 선택 규칙 통일
	- skills / agents 계층화 거버넌스와 `full-stack-master` PDTFC+ 흐름 수정
	- 세 갈래 전용 회귀: 코드 품질 / 대형 파일 정리, 문서 / 설정 / 데이터베이스 동기화, smoke / coverage / 성능 / 의존성 보안 청정 기준선
